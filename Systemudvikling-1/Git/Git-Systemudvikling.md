# Systemudvikling Hold A

Som kommende Datamatikere og Systemudviklere, så ved I allerede nu, at det er meget vigtigt at sikre såvel den røde tråd, sporbarhed samt en høj kvalitet i de artefakter I udvikler.
Det er noget vi kæmper rigtig meget med at sikre - og alligevel går det meget ofte galt.

Det vil vi søge at ændre på, ved at anvende Git, Visual Studio Code m.m. i vores arbejde med at udvikle og vedligeholde vores artefakter.
Artefakter er sådan løseligt forklaret alt det vi selv producerer eller modtager i samarbejde med kunden (Virksomheden).

Så når vi laver Business case, Interessentanalyse, Forretnings Model Lærred (FML), Aktivitets diagrammer eller BPMN'er, Use Cases, Domænemodel osv. så skal vi sikre at de oplysninger om den information vores system skal håndtere, transporteres fra det ene artefakt til det næste. Fra vores Analyse, Design til vores Implementering. Derigennem kommer sporbarheden. Rækkefølge vi udfører og fremstiller artefakterne, er det vi kalder den røde tråd.
Det er vores opdrag at sikre den røde tråd og sporbarheden!

Denne guide er ikke udtømmende, men stræber på at være temmelig detaljeret, således at I kan anvende guiden til at sikre I har de rigtige programmer installeret konfigureret på den rigtige måde samt at I har de templates som I skal bruge m.v. Krydret med nogle ting I skal huske på for at sikre, at I gør det på en god måde.

# Hvordan skal vi arbejde?
I skal anvende en editor. Jeg anbefaler at I anvender Microsoft Visual Studio Code. Visual Studio Code finde til Windows, Linux og Mac. Der er mange fordele ved, at lære at anvende Visual Studio Code. Bl.a. er det, det mest anvendte "Integrated Development Environment" (IDE).
Visual Studio Code har en desuden indbygget terminal, hvorigennem I skal anvende Git og Git kommandoer til at styre og dele jeres kode og filer. Det er det vi skal kigge nærmere på.

I Visual Studio Code skal I desuden anvende to såkaldte plugins: PlantUML og PlantUML Viewer. 


## Anbefaling
Undgå Microsoft Onedrive. Synkroniseringen kan skabe problemer for jer når I også anvender Git.
En god løsning er at anvende to foldere på jeres C drev (C:\). Her anbefaler jeg, at I laver en folder med jeres navn f.eks. Søren Meng Andersen eller initialer SMA og en anden der hedder Systemudvikling med to underfoldere Systemudvikling-1 og Systemudvikling-2.
Så har I en stuktur der passer til første og andet semester (C:\Systemudvikling\Systemudvikling-1) og senere på tredje semester (C:\Systemudvikling\Systemudvikling-2).

Dvs. så I har følgende:
* C:\Systemudvikling\Systemudvikling-1
* C:\Systemudvikling\Systemudvikling-2
* C:\SMA

Så I skal have følgende installeret. Spring over hvis I allerede har det installeret.
